9def helper_render_date(value, var_date, var_start_date, var_end_date): 

10 """ 

11 helper function to avoid duplicated code. It checks the various sub-dates of a model's date field for them being None 

12 or having values. If a field is None then check for the next, and if all are None, return '—' inditcating no value. 

13 

14 If there are values, use them as mouse overlay helptext to inform the user about the parsing result behind a written 

15 date field. 

16 

17 :param value: str : the \*_date_written (either start_date_written or end_date_written) field of an entity or relation 

18 :param var_date: datetime : either the precisely parsed date or the average in between two dates when \*_date_written is a range 

19 :param var_start_date: datetime : The sub-date of var_date, indicating the start date of the range 

20 :param var_end_date: datetime : The sub-date of var_date, indicating the end date of the range 

21 :return: html string : which has the value of the written date and the parsed dates as mouse overlay helptext 

22 """ 

23 

24 # Various if-else branches checking which of the date fields are not None and should be used 

25 

26 if var_start_date is not None and var_end_date is not None: 

27 overlay_help_text = str(var_start_date) + " - " + str(var_end_date) 

28 

29 elif var_date is not None: 

30 overlay_help_text = str(var_date) 

31 

32 else: 

33 return "—" 

34 

35 return format_html("<abbr title='" + overlay_help_text + "'>" + value + "</b>")
